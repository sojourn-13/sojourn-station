/datum/penetration_holderHolds variables important to the functioning of trace projectiles. If a trace projectile penetrates anything, it will be added to the force_penetration_on list. If penetration_store_time is more than world.time, and this datum is passed as an argument to Shoot(), the fired projectile will always penetrate anything in that list. This datum will then be qdeleted.
/proc/check_trajectory_raytrace Fires a projectile that uses the exact same targetting, trajectory, penetration, etc. logic it normally would, but invisible, and without any of the other effects, such as damage, visuals, sound, messages, etc. Returns trace.
